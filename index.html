AC自动机

发表于 2022-09-02 分类于 String

概述

AC自动机 是以 Trie 为结构，结合 KMP 思想对 Trie 上的所有节点构造失配指针构成的。
构造出失配指针后就可以用和 KMP 类似的方式进行多模式匹配
以下时用到的一些记号

$n$ —— 模式串的个数
$s_i$ —— 模式串 $(i=1,2, \dots,n)$
$N$ —— AC自动机中结点数
$M$ —— 文本串长度

构造 Trie

Trie 中的节点表示的是某个模式串的前缀，一个节点表示一个状态，而 Trie 的边就是状态的转移
形式化地说，对于若干个模式串 $s_1,s_2,\dots,s_n$，将它们构建一棵字典树后的所有状态的集合记作 $Q$。

广义 Border

在 KMP 里定义的 $Border$ 是一个模式串的相同前后缀，而 广义 $Border$ 是指在所有模式串中匹配的相同前后缀，举个例子：
模式串：ACAM、MAAC
第一个模式串的前缀 AC 和 第二个模式串的后缀 AC 匹配，因此，AC 就是这个模式串集合 $Q$ 的一个 广义 $Border$

失配指针

与 KMP 的 $next$ 指针 类似，ACAM 里的 $fail$ 指针 指的是所有模式串中最长前缀匹配当前状态的后缀（对于当前状态的最长 $Border$）
即，对于状态 $u$ 的 fail 指针指向的状态 $v$, $v$ 是 $u$ 的最长后缀 $(u, v \in Q)$
构建 $fail$ 指针和在 KMP 中构造 $next$ 指针类似：
对于 $Trie$ 中的一个结点 $u$，$u$ 的父节点是 $p$，$p$ 通过字符 $c$ 的边指向 $u$, 即 $tr[p][c] = u$

  1. 如果 其父节点的 fail 指针指向的状态的后继有 $c$，即 $tr[fail[p]][c]$ 存在, 则让 $u$ 的 $fail$ 指针指向这个后继状态，即 $fail[u] = tr[fail[p]][c]$
  2. 如果 $tr[fail[p]][c]$ 不存在，那么继续往上找 $tr[fail[fail[p]]][c]$，重复 1 的过程直到 $fail$ 指针跳到根节点
  3. 如果 $fail$ 指针跳到了根节点，就让 $fail$ 指针指向根节点
    如此就完成了$fail$ 指针的构建

Trie 图

很容易发现，上述构造 fail 指针的过程中，每匹配到一个结点就要往上不断暴力跳 $fail$ 边，比如模式串 AB，AAB，AAAB，AAAAB 和 AC 与文本串 AAAAC 匹配时，AAAAC 会先跳到 AAAC，然后再跳到 AAC， 然后才会跳到 AC。
如果失配的时候可以一步到位，就可以省去暴力跳 $fail$ 边的复杂度。因此可以在构建 $fail$ 指针时，如果 $tr[p][c]$ 不存在就令 $tr[p][c] = tr[fail[p]][c]$
这样原本的 DAG 结构的 Trie 上出现了环，这样的结构就叫做 Trie 图。

模式串匹配

$fail$ 指针构建好后就可以匹配计数了，和 kmp 不太一样的点在于，当前状态的所有的广义 $Border$ 都是需要遍历的，可以从最长的 $Border$ 一步一步往上跳直到跳到根节点。
若某个结点是该状态下最长的 $Border$，即不需要往上跳 $fail$ 边的结点，那么这个结点被直接匹配
若这个被访问的结点是被直接匹配的结点跳 $fail$ 边匹配到的，这这个结点叫可被匹配
显然这个结点被匹配到的次数 = 被直接匹配的次数 + 可被匹配的次数

$fail$ 树计数优化

一个结点只有一个 $fail$ 指针指出，可能被多个 $fail$ 指针指向。因此，$fail$ 指针构成了一个以根节点为根的 $fail$ 树，一个结点的 $fail$ 指针指向它在 $fail$ 树中的父亲。
显然，$fail$ 树中以结点 $u$ 为根的另一个结点 $v$ 被直接匹配到，那么 $u$ 在 $v$ 往上跳 $fail$ 边的时候也会被匹配到。简单来说，如果每次只给被直接匹配到的结点的数量加一，那么对于一个结点可以被匹配到的次数就是以这个结点为根的子树中所有结点被直接匹配的次数的和。
这样对于每一次匹配不再需要往上跳 $fail$ 边，只需要最后在树形dp统计一下 $fail$ 树的子树大小即可
分析一下时间复杂度： 文本串匹配的过程是 $O(M)$，$fail$ 树做树形dp的过程是 $O(N)$，因此总的复杂度是 $O(M + N)$

KMP

发表于 2022-08-01 更新于 2022-09-02 分类于 String

Border

约定：字符串 $S$ 的长度为 $n$, $[l,r]$ 的子串是 $S_{l,r}$, 前缀 $S_{1,i} = pre_i$， 后缀 $S_{n - i + 1, n} = suf_i$
定义字符串 $S$ 的一个 Border 为一个满足 $pre_i=suf_i$ 的前缀，其中 $S$ 和 $\emptyset$ 也是 $S$ 的 Border
记 $S$ 的 Border 集合为 $B(S)$，设最长的 Border 为 $LBorder$

Period（周期）

若 $0 < p \le n, S[i] = S[i + p], \forall i \in [1, n]$，则称 $p$ 为 $S$ 的一个 Period
记最小 Period 为 $per(S)$
结论：$p$ 是 $S$ 的 Period $\Leftrightarrow$ $|S|-p$ 是 $S$ 的 Border
证明：
$p$ 是 $S$ 的 Period $\Leftrightarrow S[i-p] = S[i]$
$q$ 是 $S$ 的 Border $\Leftrightarrow S[1,q]=S[n-q+1, n] \Leftrightarrow S[1] = S[n-q+1]$
令 $i=p+1$ 即可得到 $S[p+1-p]=S[n-q+1]=S[p+1]$ 即
$n-q+1=p+1$ 即 $n=p+q$

Weak Periodicity Lemma

若 $p$ 和 $q$ 是 $S$ 的 Period，$p + q \le n$, 则 $gcd(p,q)$ 也是 $S$ 的 Period
证明：
设 $p < q$, 记 $d=q-p$
若 $i > p$, 则 $i-p>0$, 故 $S[i]=S[i-p]=S[i-p+q]=S[i-d]$
若 $i \le p$, 则 $i+q\le n$, 故 $S[i]=S[i+q]=S[i+q-p]=S[i-d]$
因此, $d$ 也是 $S$ 的 Period, 更相减损, 故得证

Periodicity Lemma

若 $p, q$ 是 $S$ 的 Period，且 $p + q \le |S| + gcd(p, q)$ ，则 $gcd(p,q)$ 也是 $|S|$ 的一个Period
证明看起来非常复杂

LBorder

对于一个字符串 $p$，$p$ 的 Border 的 Border 也是 $p$ 的 Borde，因此可以通过$p[i - 1]$ 的最长 Border 来推出 $p[i]$ 的最长 Border
不妨令 $p[1, j] = p[i - j, i - 1]$ ($p[1,j]$ 为 $p[i - 1]$ 的最长 Border)，$next[j]$ 为 $p[j]$ 的最长 Border长度：

  1. 若 $p[j + 1] = p[i]$， 则：$j = j + 1，next[i] = j$
  2. 若 $p[j + 1] \ne p[i]$ ，则令 $j$ 跳到 $next[j]$ 再次进行匹配 这段代码也可以理解为： 让 $p$ 后移一位变为 $p’$，然后让 $p’$ 的前缀去匹配 $p$ 的后缀（自己匹配自己）

KMP 匹配

对于文本串 $s$ ，模式串 $p$，若 $s[i - j, i - 1] = p[1, j]$，即模式串的前缀 $j$ 已经匹配成功
如果 $s[i] \ne p[j + 1]$，那么只需要令 $j$ 跳到 $next[j]$ 再次进行匹配
如果 $s[i] = p[j + 1]$，说明可以匹配则令 $j = j + 1$ ，如果 $j = n$ 说明模式串匹配成功

例题

P3375 【模板】KMP字符串匹配

Problem - 1200E - Codeforces
将 $n$ 个字符串合并成一个字符串。合并两个字符串 $A$, $B$ 时，删除 $B$ 中与 $A$ 的后缀重合的最长前缀，比如将 “sample” 和 “please” 合并为 “samplease”
很容易想到将 $B$ 拼到 $A$ 的前面，这样形成 $BA$ 只需要找到这个字符串的最长 border 长度，合并时删去这个 border 就好了。（注意合并时候为了防止越界在中间加一个特殊字符）

然后T了， 考虑到每一次合并的 ans 比较长，而且 ans 前面的字符对答案没有贡献
比如：$abfdsarsewfvdsarreab$ 和 $aba$ 拼在一起的时候前面只需要保留 ans 的后三位，$abaeab$ 即可

当然这样还是T了
又犯了一个犯了很多次的错误 不应该直接这样拼接，换成+=就成功过掉了（再犯这个错误就紫砂）

在字符串 $s$ 中，重复删除同一个字串 $t$。直到 $t$ 不再是 $s$ 的字串为止，$|s|\le10^6$
考虑用一个栈来维护 $s$, 和 $t$ 匹配成功后就将匹配的部分弹出。这样看起来就很对，但是样例没跑过。
问题在忽略了删除后两边重新凑成 $t$ 的情况，那么用一个数组记录一下 $s[i]$ 已经匹配到的 $t$ 的位置（类似于next数组），匹配成功后弹出相应的部分，$j$ 跳到栈首位置匹配到 $t$ 的位置就可以啦
